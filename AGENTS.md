## Learned User Preferences

- Workshop Beamer slides should explain what each harness concept is, using [AgenticCodingForEconomists](https://github.com/antoniomele/AgenticCodingForEconomists) `slides/day*` decks as the style reference.
- In Beamer `itemize` slides, use `\textnormal{}` for body text after bold item titles when a leading `\textbf{}` would otherwise inherit style or color across the whole slide.
- When workshop slides or docs disagree with live Cursor product documentation, prefer the official docs and add a short “verify in your Cursor version” callout where UI or field names vary by release.
- Fix Beamer `Overfull \vbox` warnings by condensing frame content (shorter listings, fewer nested blocks) rather than relying on large `shrink` values alone.
- Append session content to `cursor_chat_recording.md` at the repo root.
- When implementing an attached plan, do not edit the plan file itself.

## Learned Workspace Facts

- Four-hour LSE PhD workshop adapting the full Agentic Coding for Economists course into one research-article harness path centered on `examples/starter_article/`.
- Canonical projector slides: `slides/workshop/workshop_slides.tex` and `slides/shared/beamer_preamble.tex`; rebuild with `npm run slides:build`.
- Five tool lanes (Codex CLI/app, Claude Code CLI/app, Cursor) share the same harness artifacts; merge lane files from `examples/{codex,codex-app,claude,claude-app,cursor}/` per `materials/tool_tracks.md`.
- Workshop delivery site is VitePress under `website/`; per-track module slices are generated by `scripts/build_site_tabs.py` into `website/docs/modules/_generated/`.
- Generated website links to repo paths must go through `rewrite_links_for_vitepress()` in `build_site_tabs.py` so template and example URLs resolve on the site (avoid `../` 404s).
- Completion criteria and timing: `docs/research_article_harness.md`, `materials/module_*.md`, and `INSTRUCTOR_RUNBOOK.md`.
